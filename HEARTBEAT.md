# HEARTBEAT.md - 心跳任务配置

本文件是 my-openclaw 项目的心跳任务执行指引。当心跳定时触发时，会读取此文件作为上下文参考。

## 心跳执行指引

### 1. 项目状态检查
- 检查 `MEMORY.md` 了解当前项目上下文
- 检查 `AGENTS.md` 了解当前 Agent 配置
- 查看今日 `memory/YYYY-MM-DD.md` 是否有未完成任务

### 2. Git 状态检查
- 检查是否有未提交的更改
- 检查是否有待 push 的提交
- 检查是否有需要 merge 的分支

### 3. 服务状态检查
- 检查 Backend 服务是否正常运行 (端口 3001)
- 检查 UI 服务是否正常运行 (端口 5173)
- 检查 Node.js 进程状态

### 4. 健康度评估
根据以上检查结果，评估项目健康度：
- 🟢 正常：所有服务运行正常，无异常
- 🟡 注意：存在未提交更改或服务波动
- 🔴 异常：服务停止或需要立即处理的问题

### 5. 执行频率建议
- 默认检查间隔：30 分钟
- 如无异常，静默执行（不打扰用户）
- 如有异常，通过钉钉机器人通知用户

## 注意事项
- 本文件是心跳任务的执行指引，不是任务清单
- 心跳服务会读取 MEMORY.md 和 HEARTBEAT.md 作为上下文
- 有重要事项时触发 heartbeat:alert 事件通知
