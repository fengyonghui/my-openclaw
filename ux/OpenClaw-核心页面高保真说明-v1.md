# 🎨 UI/UX 实现方案：OpenClaw 核心页面高保真说明

## UX 分析
- **核心信息必须前置**：项目名、当前 Chat、当前 Agent 和运行状态必须始终可见，否则用户在多项目场景中容易迷失上下文。
- **辅助信息不能压主任务流**：Session、Agent、Background Tasks 很重要，但应该作为辅助面板存在，不能挤压消息流和主内容区。
- **高保真阶段要补齐反馈层**：按钮悬停、页面空状态、错误提示、加载骨架和焦点态必须定义清楚，否则实现时容易风格漂移。

## 设计决策
- **配色**：高保真稿继续使用蓝色主色 + 中性灰工作台体系，主按钮采用高对比色，辅助模块以浅灰背景形成层次。
- **布局**：延续 App Shell 三栏结构，重点把栅格、卡片密度、标题层级和区块节奏定义清楚。
- **交互**：所有高频操作都定义 hover / active / disabled / loading；所有重要状态都定义 badge 与提示样式。
- **无障碍**：所有高保真说明都默认包含键盘焦点态、文本对比度和最小交互热区要求。

## 1. 页面一：App Shell

### 1.1 结构说明

```text
Top Bar: 64px
Sidebar: 280px（展开）/ 72px（折叠）
Main Content: 自适应剩余宽度
Context Panel: 320px
```

### 1.2 Top Bar 高保真要求

#### 左侧
- Project Switcher：宽 220~260px
- 展示当前项目名 + 简短描述或环境标签

#### 中部
- 全局搜索框：宽 320~420px
- 占位文案：`搜索 Chat、Agent、文件或命令...`

#### 右侧
- Global Status 胶囊
- 用户头像 / 设置入口

#### 样式要求
- 背景：纯色或轻微透明磨砂
- 下边框：1px 弱分割线
- 阴影：极轻，不可抢主内容区注意力

## 2. 页面二：Project Dashboard

### 2.1 页面目标

这是用户进入项目后的第一视觉落点，高保真稿重点是“概览清晰 + 可立即行动”。

### 2.2 布局建议

#### 顶部 Hero 区
- 项目名称
- 项目描述
- 当前默认 Agent
- 当前主目录
- 右侧快捷按钮：`新建 Chat` / `新建 Agent`

#### 中部信息卡区
三列栅格：
- 最近 Chats
- 热门 Agents
- 最近 Activity

#### 底部扩展区
- Running Tasks
- Quick Links
- Project Health / Summary

### 2.3 视觉细节

#### Hero 区
- 标题：24px / 700
- 描述：14px / 中灰
- 信息标签：使用轻量 badge
- 按钮：主次分明，主操作仅一个高亮色

#### 信息卡
- 卡片白底
- 16px 圆角
- 20~24px 内边距
- 标题与内容之间保持明显层次

---

## 3. 页面三：Chat 详情页

### 3.1 页面目标

这是最核心页面，高保真要求必须把“项目上下文”和“聊天工作流”自然融合。

### 3.2 布局建议

#### 顶部 Chat Header
- 面包屑：`my-openclaw / 项目化改造设计`
- 当前 Agent
- 当前模型
- 当前工作目录
- 运行状态 Badge
- 右侧可放“切换 Agent”“更多操作”

#### 消息流区域
- 用户消息：右对齐或浅色气泡
- Agent 消息：左对齐或中性背景卡片
- System 消息：更轻、更窄、带状态图标

#### 底部输入区
- 多行输入框
- 发送按钮
- 附加工具按钮（可选）
- 输入框上方可显示运行中状态 / 停止生成入口

#### 右侧 Context Panel
- Session Info
- Agent Info
- Background Tasks
- Recent Logs

### 3.3 高保真状态说明

#### 默认态
- 显示历史消息
- 输入区可用

#### 运行中
- 输入区顶部出现进度提示条
- 发送按钮切换为停止按钮
- 当前消息气泡底部出现 loading 动画

#### 错误态
- 对应消息卡片下方展示错误原因
- 提供 `重试` 按钮

#### 空状态
- 居中引导语
- 提示可输入示例任务

---

## 4. 页面四：Agents 页面

### 4.1 布局建议

- 顶部工具栏：标题 + 搜索 + 新建 Agent
- 中部卡片网格 / 列表
- 支持状态筛选、角色筛选

### 4.2 Agent 卡片字段

- 名称
- 角色
- 能力标签（skills）
- 最近使用时间
- 默认目录 / Memory Mode（弱化展示）
- 操作：编辑 / 复制 / 设为默认

### 4.3 视觉风格

- 卡片信息密度偏中等，不宜过满
- 用标签和小图标增强识别
- 操作按钮弱化到 hover 后更明显

---

## 5. 页面五：Files / Memory / Activity

### 5.1 Files
- 偏工具型布局
- 用表格或列表展示 workspace
- 主目录明显高亮

### 5.2 Memory
- 偏知识面板风格
- 建议分块：长期记忆 / Chat 摘要 / 固定规则

### 5.3 Activity
- 偏时间轴风格
- 每条事件显示：时间、操作者、动作、对象、摘要

## 7. 空状态与异常态设计

### 新项目空状态
- 主文案：`这个项目还没有内容`
- 次文案：`先创建一个 Chat 或 Agent，开始你的第一个任务。`
- CTA：`新建 Chat` / `新建 Agent`

### 无 Agent 空状态
- 推荐创建 `PM Agent` 或 `Backend Agent`

### 错误态
- 顶部轻提示 + 局部错误卡片
- 不建议全页报错覆盖，除非页面完全不可用

---

## 8. 无障碍与交互热区

- 所有按钮最小点击区域 ≥ 40x40
- 焦点态必须明显
- 状态图标需配文案
- 文本对比符合 AA
- 关键交互控件需支持键盘访问
