# OpenClaw 核心交互流程 v1

> 基于 `pm/` 目录下的产品需求整理，供 UX 评审、前端交互实现和测试用例设计使用。

---

## 1. 创建项目流程

```text
用户打开应用
 -> 点击顶部 Project Switcher
 -> 选择“新建项目”
 -> 填写项目名称、描述、根目录、默认模型、默认 Agent
 -> 点击创建
 -> 系统创建项目
 -> 跳转到 Project Dashboard
 -> 若用户选择创建首个 Chat，则进入新建 Chat 流程
```

### 关键体验点
- 创建入口必须容易发现
- 表单字段不宜过多
- 创建成功后要给明确反馈

---

## 2. 切换项目流程

```text
用户点击当前项目名
 -> 展开最近项目列表
 -> 选择目标项目
 -> Sidebar / Main Content / Context Panel 同步刷新
 -> 默认打开该项目最近访问页面
 -> 若该项目有运行中任务，在顶部状态区保持可见
```

### 关键体验点
- 切换要快
- 切换后不要丢失上下文感
- 运行中的任务不能因切换而“消失”

---

## 3. 创建 Chat 流程

```text
用户进入某项目
 -> 点击 + New Chat
 -> 填写标题
 -> 选择默认 Agent
 -> 选择工作目录 / 模型
 -> 点击创建
 -> 系统创建 Chat + 主 Session
 -> 跳转到 Chat 详情页
```

### 关键体验点
- 默认值要尽量智能
- 创建后应立即可用
- 不应要求用户理解太多底层概念

---

## 4. 在 Chat 中工作流程

```text
用户进入 Chat
 -> 查看历史消息
 -> 输入新任务
 -> Agent 开始处理
 -> 系统显示运行中状态
 -> 右侧面板更新 Session / Agent / Background Tasks
 -> 输出结果返回消息流
```

### 关键体验点
- 运行反馈要及时
- 消息流要保持主视觉中心
- 错误和重试要清楚

---

## 5. 创建 Agent 流程

```text
用户进入 Agents 页面
 -> 点击 New Agent
 -> 输入名称、角色、Prompt
 -> 选择 Skills、Tools、Memory Mode
 -> 点击创建
 -> 系统生成 Agent Template
 -> 新 Agent 出现在当前项目 Agent 列表中
```

### 关键体验点
- 第一版配置不要太重
- 模板与实例区别对用户应尽量隐去
- 常用角色可以提供预设

---

## 6. 多项目并发工作流程

```text
用户在项目 A 的 Chat 中启动任务
 -> 切换到项目 B
 -> 在项目 B 中打开另一个 Chat 并启动任务
 -> 顶部 Global Status 显示多项目运行中
 -> 用户可随时切回任意项目查看进度
```

### 关键体验点
- 并发信息集中展示
- 项目之间不能串内容
- 当前主任务与后台任务要有层级区分

---

## 7. 查看 Activity 流程

```text
用户进入 Activity 页面
 -> 查看按时间排序的项目事件流
 -> 可筛选 Chat / Session / Agent / File 类型
 -> 点击某条记录
 -> 跳转到对应详情页或定位上下文
```

### 关键体验点
- 活动流不是日志堆积
- 需要可读的摘要和筛选能力
- 应支持快速追溯到问题源头

---

## 8. Files / Workspace 管理流程

```text
用户进入 Files 页面
 -> 查看当前项目挂载目录
 -> 新增或编辑 workspace
 -> 设定主 workspace
 -> 标记某目录为只读
 -> 保存
```

### 关键体验点
- 目录边界要清晰
- 危险操作要有确认
- 当前默认目录应明显展示

---

## 9. Memory 查看流程

```text
用户进入 Memory 页面
 -> 查看项目长期记忆
 -> 查看各 Chat 摘要
 -> 定位当前项目的关键规则与上下文
```

### 关键体验点
- 不要做成技术性数据堆
- 应更像“项目知识面板”
- 内容应支持快速浏览

---

## 10. 异常与错误状态建议

### 常见错误场景
- 项目切换失败
- Chat 创建失败
- Agent 运行失败
- 文件目录不可访问
- 并发写冲突

### 设计建议
- 内联错误提示优先
- 关键失败场景提供重试按钮
- 重大冲突提供明确解释，而不是只抛底层报错

---

## 11. 结论

这套交互流程的核心是：

> **把 Project 作为一级上下文，把 Chat 作为主工作流，把 Session / Agent 作为辅助上下文。**

这样用户的认知负担最小，系统的扩展空间也最大。
